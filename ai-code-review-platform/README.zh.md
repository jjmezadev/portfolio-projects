# AI Code Review 平台

该平台用于结合 LLM、静态规则和代码上下文分析 Pull Request 与代码仓库，在合并前识别 bug、安全风险、性能问题以及技术债务。

## 问题

人工 Code Review 往往缓慢且不稳定，也容易遗漏安全、可维护性或性能问题。团队需要更早、更可执行的反馈，同时又不能显著增加流程负担。

## 价值主张

- 自动生成带严重级别和修复建议的 PR 评论。
- 覆盖安全、性能、测试和架构等问题类别。
- 可接入 GitHub、GitLab 与 CI 流程。
- 企业模式支持按仓库治理和完整追踪。

## 高层架构

- 接收 diff 与相关文件上下文。
- 采用规则、AST 信号和专用 LLM prompt 的混合分析器。
- 严重级别与置信度评分引擎。
- 自动生成评论、管理摘要和合并检查清单。

## 关键交付物

- 可配置类别的 PR 分析引擎。
- 评论与 summary checks 集成。
- 团队和仓库级质量仪表盘。
- 使用合成 PR 与匿名历史样本的评估套件。

## 路线图

### 调研阶段

- 梳理当前评审流程、支持语言与治理要求。

### 原型阶段

- 处理 diff、输出 findings，并在历史 PR 上验证精度。

### 实施阶段

- 增加 CI 阻断、反馈学习与组织级治理能力。

### 交付阶段

- 提供操作手册、策略示例和采用看板。

# 实时反欺诈引擎

该项目面向支付、金融科技与电商场景，用于在实时交易流中结合规则、异常检测、行为画像以及异步 LLM 解释能力完成风险判定。

## 问题

欺诈交易会直接造成收入损失、拒付成本和运营负担。仅依赖静态规则的系统通常会带来过多误报，而黑盒模型如果缺少解释、审计与分析师工作流，也很难在业务中真正落地。

## 价值主张

- 单笔交易亚 500ms 风险决策。
- 规则与 ML 结合，并具备可解释性。
- 面向分析师的审查队列与规则调优界面。
- 通过反馈闭环持续降低误报并提升准确率。

## 高层架构

- 通过 Kafka 或 Pub/Sub 进行流式接入。
- 规则引擎处理频率、黑名单、异常地理位置和金额突增。
- 异常检测模型输出行为与上下文风险分数。
- 异步 LLM 层生成命中原因解释。
- 审计日志与运营可视化面板。

## 关键交付物

- 带欺诈标签的合成数据集。
- 可回测的反欺诈规则库。
- 实时决策流水线与分析师审查队列。
- 准确率、召回率、延迟和吞吐指标体系。

## 路线图

### 调研阶段

- 明确欺诈模式、SLA 和评估方案。

### 原型阶段

- 构建规则引擎、流式处理层和异常检测基线。

### 实施阶段
